## 思路  

- 重试机制
- 生成幂等号的方式  
    - 集中生成然后发布
    - 根据约定由使用方生成
- 幂等号
    - 获取全局一致的无状态幂等号
    - 获取和业务相关的幂等号
- 验证幂等号
- 在统一的地方接入幂等框架
- 低延迟  
- 容错强，框架失败后不影响正常的流程  
    - 异常处理情况：  
        - 发送请求失败时异常，幂等号暂时还未存入，不影响正常幂等性  
        - 业务逻辑处理异常  
            - 业务代码异常  
            - 业务系统宕机  
            - 幂等框架异常  
        - 返回执行结果异常，幂等号已经存入，不影响正常的幂等性   
- 将异常产生时是否重试交付给使用者决定  
- 服务降级策略
